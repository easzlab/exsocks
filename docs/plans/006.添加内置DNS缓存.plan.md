### dns-cache ###
为 DNS 解析正常默认缓存300秒，增加"负缓存"（negative caching）机制：DNS 解析失败时缓存失败结果 30 秒，避免短时间内对同一不可达域名反复发起 DNS 查询。


# P2 DNS 解析缓存方案（增强：负缓存）

## 背景

当前 DNS 缓存仅缓存解析成功的结果。当域名解析失败时（如域名不存在、DNS 服务器不可达），每次连接请求都会重新发起 DNS 查询，造成不必要的延迟和 DNS 服务器压力。

本次增强引入"负缓存"（negative caching）机制：
- **解析成功**：缓存 300 秒（`dns_cache_ttl`，已有）
- **解析失败**：缓存 30 秒（`dns_cache_negative_ttl`，新增），在此期间直接返回缓存的错误，不再重复查询

## Proposed Changes

### DNS 缓存模块增强

#### [MODIFY] [dns_cache.rs](file:///Users/jin.gjm/code/rust/exsocks/src/dns_cache.rs)

核心改动：

1. **`DnsCacheEntry` 增加成功/失败状态**：

```diff
  struct DnsCacheEntry {
-     addrs: Vec<SocketAddr>,
+     result: DnsResult,
      created_at: Instant,
  }

+ enum DnsResult {
+     /// 解析成功，包含地址列表
+     Success(Vec<SocketAddr>),
+     /// 解析失败，包含错误描述
+     Failure(String),
+ }
```

2. **`DnsCache` 增加 `negative_ttl` 字段**：

```diff
  pub struct DnsCache {
      cache: DashMap<String, DnsCacheEntry>,
      ttl: Duration,
+     negative_ttl: Duration,
      max_entries: usize,
  }
```

3. **`DnsCache::new()` 签名变更**：

```diff
- pub fn new(ttl: Duration, max_entries: usize) -> Self
+ pub fn new(ttl: Duration, negative_ttl: Duration, max_entries: usize) -> Self
```

4. **`resolve()` 逻辑增强**：
   - 缓存命中时，根据 `DnsResult` 类型选择对应的 TTL 检查（成功用 `ttl`，失败用 `negative_ttl`）
   - DNS 解析失败时，将失败结果缓存（使用 `negative_ttl`）
   - 命中负缓存时，直接返回 `SocksError::InvalidAddress` 错误

5. **`evict_expired()` 增强**：根据条目类型使用对应的 TTL 判断是否过期
