### 支持目标地址规则管控 ###
为 exsocks SOCKS5 代理服务器添加目标地址（IP/域名）规则管控功能，支持 DOMAIN、DOMAIN-SUFFIX、IPCIDR（IPv4+IPv6）三种匹配类型，采用紧凑数组配置格式（含 opt1 位标志和 opt2 预留扩展字段），分类索引数据结构优化热路径匹配性能，所有规则按配置顺序 first-match-wins。


# 支持目标地址规则管控

为 exsocks 添加目标地址规则管控功能，在 SOCKS5 请求解析完成后、连接目标之前，根据配置的规则对目标地址（IP/域名）和端口进行匹配检查，决定是放行还是阻止。

## User Review Required

> [!IMPORTANT]
> **默认行为为 BLOCK**：当目标地址不匹配任何规则时，默认阻止连接。这意味着启用目标规则管控后，必须显式配置 PASS 规则才能允许连接。

> [!WARNING]
> **DOMAIN-SUFFIX 匹配逻辑**：`baidu.com` 将匹配 `www.baidu.com`、`tieba.baidu.com`，同时也精确匹配 `baidu.com` 本身。但不会匹配 `notbaidu.com`（必须以 `.` 分隔或完全相等）。

> [!IMPORTANT]
> **性能设计**：规则可能有上千条，匹配在每个连接的热路径上执行。采用分类索引数据结构，将 O(N) 线性扫描优化为 O(1)/O(L) 查找，同时保持 first-match-wins 语义。

> [!NOTE]
> **配置格式**：采用紧凑数组格式 `[type, value, port_start, port_end, action, opt1, opt2]`，其中 `opt1` 为 8 位二进制字符串（位标志），`opt2` 为浮点数。本期只实现 `opt1` 的 bit0（日志开关），其余位和 `opt2` 作为未来扩展预留。

## Proposed Changes

### 配置格式设计

采用紧凑数组格式，每条规则为一个 5-7 元素数组：

```yaml
target_rules:
  # [type, value, port_start, port_end, action, opt1, opt2]
  - [DOMAIN-SUFFIX, baidu.com, 0, 65535, PASS, 1, 0]
  - [IPCIDR, 127.0.0.1/32, 0, 65535, PASS, 1, 0]
  - [IPCIDR, 0.0.0.0/0, 0, 65535, BLOCK, 1, 0]
```

**opt1 位标志定义（0-255 整数，从右到左）：**

| 位 | 值 | 含义 | 本期实现 |
|----|------|------|---------|
| bit0 | `1` | 记录规则 action 日志 | 是 |
| bit1 | `2` | 限制最大并发数（opt2 为具体数值） | 预留 |
| bit2 | `4` | 限制单连接最大带宽（opt2 为具体数值） | 预留 |
| bit3-7 | - | 保留 | - |

**opt2**：浮点数，含义由 opt1 的位标志决定。本期不使用，固定为 `0`。

**解析容错**：`opt1` 和 `opt2` 为可选字段，省略时分别默认为 `0` 和 `0.0`。支持 5 元素（省略 opt1/opt2）、6 元素（省略 opt2）、7 元素（完整）三种写法。

### 高性能匹配引擎设计

#### 核心思路：分类索引 + 全局优先级

原始规则按配置文件顺序赋予优先级（越靠前优先级越高，即 `priority = 0, 1, 2, ...`）。将规则按类型分类存储到不同的索引结构中，匹配时查询相关索引，取优先级最高（数值最小）的命中规则。

**所有类型的规则共享同一个全局优先级序列**，确保 first-match-wins 语义与配置文件顺序完全一致。

```
┌─────────────────────────────────────────────────┐
│                 TargetRuleSet                    │
│                                                  │
│  ┌──────────────────┐  ┌──────────────────────┐  │
│  │  DomainIndex     │  │  CidrRules           │  │
│  │  (HashMap)       │  │  (Vec<CidrEntry>     │  │
│  │                  │  │   保持配置顺序)       │  │
│  │  "baidu.com"→[r] │  │                      │  │
│  │  "google.com"→[r]│  │  10.0.0.0/8   →[r]  │  │
│  │  ...             │  │  192.168.0.0/16→[r]  │  │
│  └──────────────────┘  │  2001:db8::/32 →[r]  │  │
│                        │  0.0.0.0/0    →[r]  │  │
│  ┌──────────────────┐  └──────────────────────┘  │
│  │  SuffixIndex     │                            │
│  │  (HashMap)       │                            │
│  │                  │                            │
│  │  "baidu.com"→[r] │                            │
│  │  "google.com"→[r]│                            │
│  └──────────────────┘                            │
└─────────────────────────────────────────────────┘
```

**各类型的索引策略：**

| 类型 | 数据结构 | 查找复杂度 | 说明 |
|------|----------|-----------|------|
| DOMAIN | `HashMap<String, Vec<PrioritizedRule>>` | O(1) | key 为小写域名，精确查找 |
| DOMAIN-SUFFIX | `HashMap<String, Vec<PrioritizedRule>>` | O(L)，L=域名层级数 | key 为后缀，逐级拆分域名查找 |
| IPCIDR | `Vec<CidrEntry>` 保持配置顺序 | O(K)，K=CIDR规则数 | 按配置顺序遍历，命中第一条即 early return |

**DOMAIN-SUFFIX 查找算法：**

对于请求域名 `a.b.c.d.com`，依次查找：
1. `a.b.c.d.com`（完全匹配）
2. `b.c.d.com`
3. `c.d.com`
4. `d.com`
5. `com`

每次查找都是 HashMap O(1)，总共最多查找域名层级数次（通常 3-5 次）。

**IPCIDR 匹配策略：**

CIDR 规则**保持配置文件中的原始顺序**，匹配时从上到下遍历。由于 IP 请求不会匹配 DOMAIN/DOMAIN-SUFFIX 规则，无需跨类型比较优先级，**命中第一条即 early return**（first-match-wins）。同时支持 IPv4 和 IPv6 CIDR：
- IPv4 示例：`10.0.0.0/8`、`192.168.0.0/16`、`0.0.0.0/0`
- IPv6 示例：`2001:db8::/32`、`::1/128`、`::/0`
- IPv4-mapped IPv6 地址（`::ffff:x.x.x.x`）自动规范化为 IPv4 后匹配

**匹配流程：**

```
check(address, port):
  1. 根据 address 类型决定查询哪些索引：
     - Domain → 查 domain_index + suffix_index，取全局优先级最高的
     - IPv4/IPv6 → 查 cidr_rules，命中第一条即返回（early return）
  2. 如果无命中规则，返回默认 BLOCK
```

> [!TIP]
> **性能分析**（假设 1000 条规则）：
> - **DOMAIN 查找**：HashMap O(1)，无论规则数量多少
> - **DOMAIN-SUFFIX 查找**：最多 5 次 HashMap 查找（典型域名层级），与规则数量无关
> - **IPCIDR 查找**：O(K) 最差，但 early return 后平均远小于 K。按配置顺序遍历，命中第一条即停止
> - **总体**：对于域名请求（占绝大多数流量），匹配复杂度从 O(N) 降低到 O(L)（L 为域名层级数，通常 3-5）

```rust
/// 规则匹配类型
#[derive(Debug, Clone, PartialEq)]
pub enum RuleType {
    Domain,        // 精确域名匹配
    DomainSuffix,  // 域名后缀匹配
    IpCidr,        // IP/CIDR 匹配（IPv4 + IPv6）
}

/// 规则动作
#[derive(Debug, Clone, Copy, PartialEq)]
pub enum RuleAction {
    Pass,   // 放行
    Block,  // 阻止
}

/// opt1 位标志常量
pub const OPT_LOG: u8 = 0b0000_0001;              // bit0: 记录日志
pub const OPT_MAX_CONCURRENCY: u8 = 0b0000_0010;  // bit1: 最大并发数（预留）
pub const OPT_MAX_BANDWIDTH: u8 = 0b0000_0100;    // bit2: 最大带宽（预留）

/// 单条目标规则（解析后）
#[derive(Debug, Clone)]
pub struct TargetRule {
    pub rule_type: RuleType,
    pub value: String,
    pub port_start: u16,
    pub port_end: u16,
    pub action: RuleAction,
    pub opt_flags: u8,   // opt1 解析后的位标志
    pub opt_value: f64,  // opt2 数值
}

/// 配置文件根结构
#[derive(Debug, Deserialize)]
struct TargetRulesConfig {
    #[serde(default)]
    target_rules: Vec<Vec<serde_yaml::Value>>,
}
```

**数组格式自定义解析逻辑（含输入校验）：**

每条规则为 `Vec<serde_yaml::Value>`，手动解析 5-7 个元素：

```rust
fn parse_rule_array(arr: &[serde_yaml::Value], index: usize) -> Result<TargetRule, SocksError> {
    // 元素数量校验：最少 5 个，最多 7 个
    if arr.len() < 5 || arr.len() > 7 {
        return Err(SocksError::TargetRulesConfig(
            format!("Rule #{}: expected 5-7 elements, got {}", index + 1, arr.len())
        ));
    }

    // type 解析
    let rule_type = match arr[0].as_str().unwrap_or("") {
        "DOMAIN" => RuleType::Domain,
        "DOMAIN-SUFFIX" => RuleType::DomainSuffix,
        "IPCIDR" => RuleType::IpCidr,
        other => return Err(SocksError::TargetRulesConfig(
            format!("Rule #{}: unknown type '{}'", index + 1, other)
        )),
    };

    // value 解析（容错：非字符串类型尝试 to_string 转换）
    let value = match arr[1].as_str() {
        Some(s) => s.to_string(),
        None => {
            // 某些 YAML 值可能被解析为非字符串（如纯数字），尝试转换
            let s = format!("{}", serde_yaml::to_string(&arr[1]).unwrap_or_default().trim());
            if s.is_empty() || s == "~" || s == "null" {
                return Err(SocksError::TargetRulesConfig(
                    format!("Rule #{}: value field is empty or null, \
                             use quotes for special values like \"::1/128\"", index + 1)
                ));
            }
            s
        }
    };

    // value 非空校验
    if value.is_empty() {
        return Err(SocksError::TargetRulesConfig(
            format!("Rule #{}: value field cannot be empty", index + 1)
        ));
    }

    // port 解析
    let port_start = arr[2].as_u64().unwrap_or(0) as u16;
    let port_end = arr[3].as_u64().unwrap_or(65535) as u16;

    // port_start <= port_end 校验
    if port_start > port_end {
        return Err(SocksError::TargetRulesConfig(
            format!("Rule #{}: port_start ({}) > port_end ({})", index + 1, port_start, port_end)
        ));
    }

    // action 解析
    let action = match arr[4].as_str().unwrap_or("") {
        "PASS" => RuleAction::Pass,
        "BLOCK" => RuleAction::Block,
        other => return Err(SocksError::TargetRulesConfig(
            format!("Rule #{}: unknown action '{}'", index + 1, other)
        )),
    };

    // opt1: 0-255 整数（位标志），默认 0
    let opt_flags = if arr.len() > 5 {
        let opt1_val = arr[5].as_u64().ok_or_else(|| {
            SocksError::TargetRulesConfig(
                format!("Rule #{}: invalid opt1: expected integer 0-255", index + 1)
            )
        })?;
        if opt1_val > 255 {
            return Err(SocksError::TargetRulesConfig(
                format!("Rule #{}: invalid opt1 '{}': expected integer 0-255", index + 1, opt1_val)
            ));
        }
        opt1_val as u8
    } else {
        0
    };

    // opt2: 浮点数，默认 0.0
    let opt_value = if arr.len() > 6 {
        arr[6].as_f64().unwrap_or(0.0)
    } else {
        0.0
    };

    Ok(TargetRule {
        rule_type, value, port_start, port_end, action, opt_flags, opt_value,
    })
}
```

**编译逻辑（`TargetRuleSet::compile`）：**

```rust
impl TargetRuleSet {
    pub fn compile(rules: Vec<TargetRule>) -> Result<Self, SocksError> {
        let total_rules = rules.len();
        let mut domain_index: HashMap<String, Vec<PrioritizedRule>> = HashMap::new();
        let mut suffix_index: HashMap<String, Vec<PrioritizedRule>> = HashMap::new();
        let mut cidr_rules: Vec<CidrEntry> = Vec::new();

        for (priority, rule) in rules.into_iter().enumerate() {
            let pr = PrioritizedRule {
                priority,
                port_start: rule.port_start,
                port_end: rule.port_end,
                action: rule.action,
                opt_flags: rule.opt_flags,
                opt_value: rule.opt_value,
            };

            match rule.rule_type {
                RuleType::Domain => {
                    let key = rule.value.to_ascii_lowercase();
                    domain_index.entry(key).or_default().push(pr);
                }
                RuleType::DomainSuffix => {
                    let key = rule.value.to_ascii_lowercase();
                    suffix_index.entry(key).or_default().push(pr);
                }
                RuleType::IpCidr => {
                    // IpNet 自动支持 IPv4 和 IPv6 CIDR
                    let network: IpNet = rule.value.parse().map_err(|e| {
                        SocksError::TargetRulesConfig(format!(
                            "Invalid CIDR '{}': {}", rule.value, e
                        ))
                    })?;
                    cidr_rules.push(CidrEntry { network, rule: pr });
                }
            }
        }

        // CIDR 规则保持配置文件中的原始顺序，不排序
        // 匹配时按顺序遍历，first-match-wins + early return

        Ok(Self { domain_index, suffix_index, cidr_rules, total_rules })
    }
}
```

**匹配逻辑（`TargetRuleSet::check`）：**

```rust
impl TargetRuleSet {
    pub fn check(&self, address: &Address, port: u16) -> MatchResult {
        let best = match address {
            Address::Domain(domain) => self.match_domain(domain, port),
            Address::IPv4(ip) => self.match_cidr(&IpAddr::V4(*ip), port),
            Address::IPv6(ip) => self.match_cidr(&IpAddr::V6(*ip), port),
        };

        match best {
            Some(rule) => MatchResult {
                allowed: rule.action == RuleAction::Pass,
                log: rule.opt_flags & OPT_LOG != 0,
                opt_flags: rule.opt_flags,
                opt_value: rule.opt_value,
            },
            // 默认 BLOCK，不记录日志
            None => MatchResult {
                allowed: false, log: false, opt_flags: 0, opt_value: 0.0,
            },
        }
    }

    /// 域名匹配：查询 domain_index 和 suffix_index，取全局优先级最高的
    fn match_domain(&self, domain: &str, port: u16) -> Option<&PrioritizedRule> {
        let lower = domain.to_ascii_lowercase();
        let mut best: Option<&PrioritizedRule> = None;

        // 1. 精确域名匹配（HashMap O(1)）
        if let Some(rules) = self.domain_index.get(&lower) {
            for r in rules {
                if r.port_matches(port) && r.is_higher_priority_than(best) {
                    best = Some(r);
                }
            }
        }

        // 2. 后缀匹配：逐级拆分域名（O(L)，L=域名层级数）
        //    对于 "a.b.c.com"，依次查找 "a.b.c.com", "b.c.com", "c.com", "com"
        let mut search = lower.as_str();
        loop {
            if let Some(rules) = self.suffix_index.get(search) {
                for r in rules {
                    if r.port_matches(port) && r.is_higher_priority_than(best) {
                        best = Some(r);
                    }
                }
            }
            match search.find('.') {
                Some(pos) => search = &search[pos + 1..],
                None => break,
            }
        }

        best
    }

    /// CIDR 匹配：按配置顺序遍历，命中第一条即 early return
    ///
    /// 由于 IP 请求不会匹配 DOMAIN/DOMAIN-SUFFIX 规则，无需跨类型比较优先级，
    /// cidr_rules 保持配置顺序，第一个匹配的就是优先级最高的，直接返回。
    fn match_cidr(&self, ip: &IpAddr, port: u16) -> Option<&PrioritizedRule> {
        let normalized = normalize_ip(*ip);

        for entry in &self.cidr_rules {
            if entry.network.contains(&normalized) && entry.rule.port_matches(port) {
                return Some(&entry.rule);  // early return: first-match-wins
            }
        }

        None
    }

    pub fn rule_count(&self) -> usize {
        self.total_rules
    }
}

impl PrioritizedRule {
    #[inline]
    fn port_matches(&self, port: u16) -> bool {
        port >= self.port_start && port <= self.port_end
    }

    #[inline]
    fn is_higher_priority_than(&self, other: Option<&PrioritizedRule>) -> bool {
        match other {
            None => true,
            Some(o) => self.priority < o.priority,
        }
    }
}
```

**`normalize_ip` 函数**：将 IPv4-mapped IPv6 地址（`::ffff:x.x.x.x`）规范化为纯 IPv4 地址。

```rust
if let Some(trc) = &target_rule_control {

    if result.log {
        if result.allowed {
            info!(target = %request.address, port = request.port, "Target PASS by rule");

            request.address.to_string(), request.port,
        ));
    }
}
```

```yaml
# action: PASS | BLOCK
# opt1: 0-255 整数（位标志），可省略，默认 0
#   bit0 (1): 记录规则 action 日志
#   bit1 (2): 限制最大并发数，opt2 为具体数值（预留）
#   bit2 (4): 限制单连接最大带宽，opt2 为具体数值（预留）
# opt2: 浮点数，含义由 opt1 决定，可省略，默认 0
#
# 规则按从上到下顺序匹配，命中第一条即停止（first-match-wins）
# 未命中任何规则时，默认阻止（BLOCK）

target_rules:
  # 允许访问 baidu.com 及其所有子域名，记录日志
  - [DOMAIN-SUFFIX, baidu.com, 0, 65535, PASS, 1, 0]

  # 允许访问 example.com 精确域名的 HTTPS 端口
  - [DOMAIN, example.com, 443, 443, PASS]

  # 允许访问本地回环地址（IPv4），记录日志
  - [IPCIDR, 127.0.0.1/32, 0, 65535, PASS, 1, 0]

  # 允许访问内网 10.0.0.0/8 网段
  - [IPCIDR, 10.0.0.0/8, 0, 65535, PASS]

  # 允许访问 IPv6 回环地址
  - [IPCIDR, "::1/128", 0, 65535, PASS]

  # 允许访问 IPv6 内网地址段
  - [IPCIDR, "fd00::/8", 0, 65535, PASS]

  # 阻止所有其他 IPv4，记录日志
  - [IPCIDR, 0.0.0.0/0, 0, 65535, BLOCK, 1, 0]

  # 阻止所有其他 IPv6，记录日志
  - [IPCIDR, "::/0", 0, 65535, BLOCK, 1, 0]
```
