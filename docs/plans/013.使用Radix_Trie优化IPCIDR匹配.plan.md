# IPCIDR 规则匹配 Radix Trie 优化

将 `target_rules.rs` 中 IPCIDR 规则的匹配数据结构从 `Vec<CidrEntry>` 线性遍历（O(K)）替换为基于 `prefix-trie` crate 的 Radix Trie（O(log n)），利用 `cover_values` API 遍历所有匹配前缀并取最小 priority，实现高效查找同时保持 first-match-wins 语义。

**2. 替换 `TargetRuleSet` 中的 CIDR 存储结构**

> [!IMPORTANT]
> IPv4/IPv6 分离是必要的，因为 `PrefixMap` 的泛型参数 `P: Prefix` 要求同一棵树中所有前缀类型一致。`prefix-trie` 提供了 `joint::JointPrefixMap` 可以同时存储 IPv4 和 IPv6，但为了保持代码清晰和控制力，选择手动分离。

**3. 修改 `compile` 方法**

```diff
-    let mut cidr_rules: Vec<CidrEntry> = Vec::new();
+    let mut cidr_trie_v4: PrefixMap<Ipv4Net, PrioritizedRule> = PrefixMap::new();
+    let mut cidr_trie_v6: PrefixMap<Ipv6Net, PrioritizedRule> = PrefixMap::new();
```

在 `RuleType::IpCidr` 分支中，按 IPv4/IPv6 分别插入对应的 Trie：

```rust
RuleType::IpCidr => {
    let network: IpNet = rule.value.parse().map_err(|e| {
        SocksError::TargetRulesConfig(format!(
            "Invalid CIDR '{}': {}",
            rule.value, e
        ))
    })?;
    match network {
        IpNet::V4(v4net) => {
            // 同一 CIDR 可能配置多次（不同端口范围），
            // 但 PrefixMap 同一 key 只能存一个 value。
            // 由于 CIDR + 端口范围组合可能不同，需要存 Vec。
            // 但实际场景中同一 CIDR 重复配置极少，
            // 这里用 entry API 处理：如果已存在且新规则 priority 更高则不替换。
            // 注意：同一 CIDR 不同端口范围的情况需要特殊处理，
            // 改为存储 Vec<PrioritizedRule>。
            cidr_trie_v4
                .entry(v4net)
                .or_insert(Vec::new())
                .push(pr);
        }
        IpNet::V6(v6net) => {
            cidr_trie_v6
                .entry(v6net)
                .or_insert(Vec::new())
                .push(pr);
        }
    }
}
```

> [!WARNING]
> **同一 CIDR 不同端口范围的处理**：`PrefixMap` 的同一个 key 只能存一个 value。如果同一个 CIDR（如 `10.0.0.0/8`）配置了多条规则（不同端口范围），需要将 value 类型改为 `Vec<PrioritizedRule>`。因此 Trie 的类型应为 `PrefixMap<Ipv4Net, Vec<PrioritizedRule>>`。

**4. 修改 `match_cidr` 方法**

利用 `prefix-trie` 的 `cover_values` API 遍历所有覆盖目标 IP 的前缀节点，取 priority 最小且端口匹配的规则：

```rust
fn match_cidr(&self, ip: &IpAddr, port: u16) -> Option<&PrioritizedRule> {
    let normalized = normalize_ip(*ip);

    match normalized {
        IpAddr::V4(v4) => {
            let host_prefix = Ipv4Net::new(v4, 32).unwrap();
            let mut best: Option<&PrioritizedRule> = None;
            // cover_values 遍历所有覆盖 host_prefix 的前缀节点
            // 即所有 contains(v4) 为 true 的 CIDR 规则
            for rules in self.cidr_trie_v4.cover_values(&host_prefix) {
                for r in rules {
                    if r.port_matches(port) && r.is_higher_priority_than(best) {
                        best = Some(r);
                    }
                }
            }
            best
        }
        IpAddr::V6(v6) => {
            let host_prefix = Ipv6Net::new(v6, 128).unwrap();
            let mut best: Option<&PrioritizedRule> = None;
            for rules in self.cidr_trie_v6.cover_values(&host_prefix) {
                for r in rules {
                    if r.port_matches(port) && r.is_higher_priority_than(best) {
                        best = Some(r);
                    }
                }
            }
            best
        }
    }
}
```

> [!NOTE]
> **`cover_values` 语义**：给定一个 `/32`（或 `/128`）的主机前缀，`cover_values` 返回所有**包含**该主机地址的 CIDR 前缀的值。这等价于遍历 Trie 从根到叶路径上所有带值的节点，复杂度 O(W)（W = 32 或 128），与规则数量 K 无关。
