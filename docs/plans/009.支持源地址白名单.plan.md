# 支持客户端源地址白名单

为 exsocks 增加客户端源地址验证功能。通过独立的 `client-rules.yaml` 配置文件定义 `client_rules` CIDR 白名单规则，在主配置中增加开关控制是否启用，支持配置文件热加载。

## 设计决策

> [!IMPORTANT]
> **白名单模式**：只有 IP 命中 `client_rules` 列表中的 CIDR 规则才允许连接，否则一律拒绝。
>
> **空规则处理**：当 `client_rules` 列表为空时，拒绝所有连接（白名单为空即无人可访问，符合最小权限原则）。
>
> **IPv4-mapped IPv6 规范化**：客户端通过 IPv6 socket 连接时，IPv4 地址会以 `::ffff:x.x.x.x` 形式出现，`is_allowed()` 方法在匹配前将其规范化为纯 IPv4 地址，确保 IPv4 CIDR 规则对此类连接有效。
>
> **静默拒绝**：被白名单拒绝的连接直接关闭 TCP 连接，不发送任何 SOCKS5 协议响应。这是有意的安全设计——不向未授权客户端暴露任何协议信息，避免被探测识别为 SOCKS5 代理。
>
> **提前检查**：白名单检查在 `accept()` 之后、`tokio::spawn` 之前的主循环中执行，而非在 `handle_connection` 内部。这样被拒绝的连接不会消耗 `ConnectionLimiter` 许可，也不会创建额外的 tokio task，在高频恶意连接场景下减少资源开销。

### 核心模块实现

#### [NEW] [access.rs](file:///Users/jin.gjm/code/rust/exsocks/src/access.rs)

**数据结构**：

```rust
/// client-rules.yaml 的反序列化结构
#[derive(Debug, Deserialize)]
struct ClientRulesConfig {
    #[serde(default)]
    client_rules: Vec<String>, // CIDR 字符串，加载后解析为 IpNet
}

/// 白名单规则集（不可变值对象，通过 ArcSwap 原子替换）
pub struct AccessRules {
    /// 白名单 CIDR 列表，按前缀长度降序排列（最长前缀优先）
    allowed: Vec<IpNet>,
}

/// 线程安全的访问控制器，支持热加载
pub struct AccessControl {
    rules: ArcSwap<AccessRules>,
    path: PathBuf,
}
```

**核心方法**：

```rust
/// 检查 IP 是否在白名单中
/// - 命中 client_rules 列表 → 放行
/// - 未命中（含列表为空）→ 拒绝
pub fn is_allowed(&self, ip: IpAddr) -> bool {
    let ip = normalize_ip(ip); // 规范化 IPv4-mapped IPv6
    self.allowed.iter().any(|net| net.contains(&ip))
}
```

**性能优化**：
- 规则列表保持配置文件中的原始顺序，从上往下依次匹配。纯白名单语义下规则之间无优先级冲突（命中任意一条即放行），无需按前缀长度排序
- `is_allowed()` 直接在 `AccessRules` 上调用，避免额外间接层

**`watch()` 实现**：完全参考 `UserStore::watch()` 的模式（`Arc<Self>` 接收者、500ms 防抖、notify channel 桥接），保持代码风格一致。

**YAML 格式**：

```yaml
client_rules:
  - 10.0.0.0/8
  - 192.168.0.0/16
  - 127.0.0.1/32
```

### 服务器集成

#### [MODIFY] [server.rs](file:///Users/jin.gjm/code/rust/exsocks/src/server.rs)

**初始化阶段**（`run_with_listener`，与 `UserStore` 初始化并列）：

```rust
let access_control = if config.access_enabled {
    let ac = Arc::new(
        AccessControl::load(&config.access_file)
            .map_err(|e| SocksError::AccessConfig(format!("Failed to load access config: {}", e)))?,
    );
    let _watcher = ac.watch().map_err(|e| {
        SocksError::AccessConfig(format!("Failed to start access config watcher: {}", e))
    })?;
    let watcher_token = cancel_token.clone();
    tokio::spawn(async move {
        watcher_token.cancelled().await;
        drop(_watcher);
    });
    info!(path = %config.access_file.display(), "Access control (whitelist) enabled with hot-reload");
    Some(ac)
} else {
    info!("Access control disabled, accepting all source addresses");
    None
};
```

**连接处理阶段**（主循环中，`accept()` 之后、`tokio::spawn` 之前）：

```rust
// 白名单检查在 accept 后立即执行，避免为被拒绝的连接创建 tokio task 和消耗 permit
if let Some(ac) = &access_control {
    if !ac.rules().is_allowed(peer_addr.ip()) {
        warn!(ip = %peer_addr.ip(), "Connection blocked by whitelist");
        drop(socket);
        continue;
    }
}
```

> [!NOTE]
> 被拒绝的连接直接关闭 TCP 连接（`drop(socket)`），不发送任何 SOCKS5 协议响应，也不消耗 `ConnectionLimiter` 许可。
