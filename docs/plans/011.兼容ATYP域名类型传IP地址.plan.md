# 兼容 ATYP 域名类型传 IP 地址

当不规范的 SOCKS5 客户端在 ATYP 字段使用 `0x03`（域名类型），但 `DST.ADDR` 实际发送的是 IP 地址字符串时，当前实现会将其解析为 `Address::Domain("x.x.x.x")`，导致目标地址规则匹配走域名分支而非 IPCIDR 分支，造成规则匹配失败。
